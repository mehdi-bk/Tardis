## Understanding `tardis_eda.ipynb` - Step by Step

This notebook does two main things: **cleans the messy data** and **creates visualizations** to understand train delays.

### Part 1: Data Cleaning

#### Step 1: Load the Data
```python
df = pd.read_csv("project_dataset.csv", sep=";", decimal=",")
```
- Loads the CSV file into a pandas DataFrame (think of it as an Excel spreadsheet in Python)
- `sep=";"` means columns are separated by semicolons (not commas)
- `decimal=","` means numbers use commas for decimals (European format: 3,14 instead of 3.14)

#### Step 2: Inspect the Data
```python
print(df.info())
print(df.isnull().sum())
```
- `df.info()` shows column names, data types, and how many non-null values exist
- `df.isnull().sum()` counts missing values in each column
- This helps us understand what we're working with

#### Step 3: Drop Useless Columns
```python
columns_to_drop = ["Commentaire annulations", ...]
df = df.drop(columns=columns_to_drop)
```
- Removes comment columns that are mostly empty (11,000+ missing values out of 12,000 rows)
- These columns don't help with analysis

#### Step 4: Remove Duplicates
```python
df = df.drop_duplicates()
```
- Finds and removes 174 duplicate rows
- Duplicates can skew our analysis, so we remove them

#### Step 5: Standardize Text
```python
df[col] = df[col].str.strip().str.upper()
```
- `.strip()` removes extra spaces: `" Paris "` → `"Paris"`
- `.upper()` converts to uppercase: `"Paris"` → `"PARIS"`
- This ensures "Paris", "PARIS", and " paris " are all treated the same

#### Step 6: Fix Station Names
```python
station_replacements = {
    "ST MALO": "SAINT MALO",
    "BORDEAUX ST JEAN": "BORDEAUX SAINT JEAN",
    ...
}
df["Gare de départ"] = df["Gare de départ"].replace(station_replacements)
```
- The dataset has inconsistent station names (some use "ST", others use "SAINT")
- We standardize them so "ST MALO" and "SAINT MALO" are treated as the same station
- Also replaces invalid entries like "0" with `NaN` (missing value)

#### Step 7: Convert Data Types
```python
for col in numeric_columns:
    df[col] = pd.to_numeric(df[col], errors="coerce")
```
- Many numeric columns are stored as text (strings) in the CSV
- `pd.to_numeric()` converts them to actual numbers so we can do math
- `errors="coerce"` means: if conversion fails, replace with `NaN` instead of crashing

#### Step 8: Parse Dates
```python
def parse_date(date_str):
    # Handles formats like "2018-01", "01-2018", "2018 01", etc.
    ...
df["Date"] = df["Date"].apply(parse_date)
```
- The Date column has inconsistent formats (some "2018-01", others "01-2018")
- This custom function handles all formats and converts them to proper datetime objects
- Proper dates let us extract year/month and do time-based analysis

#### Step 9: Feature Engineering
```python
df["year"] = df["Date"].dt.year
df["month"] = df["Date"].dt.month
df["cancellation_rate"] = df["Nombre de trains annulés"] / df["Nombre de circulations prévues"]
```
- **Feature engineering** = creating new useful columns from existing data
- Extract `year` and `month` from the Date column for time-based analysis
- Calculate `cancellation_rate` = (cancelled trains) / (scheduled trains)

#### Step 10: Handle Missing Values
```python
df[col] = df[col].fillna(df[col].median())
df = df.dropna(subset=["Gare de départ", "Gare d'arrivée", "Date"])
```
- For numeric columns: fill missing values with the median (middle value)
- For critical columns (stations, dates): drop rows that are still missing
- This ensures we have complete data for analysis

#### Step 11: Save Cleaned Data
```python
df.to_csv("cleaned_dataset.csv", index=False)
```
- Saves the cleaned DataFrame to a new CSV file
- `index=False` means don't save row numbers as a column
- This cleaned file can be used by other notebooks/scripts

### Part 2: Data Visualization

#### Visualization 1: Distribution of Delays
```python
sns.histplot(df["Retard moyen de tous les trains à l'arrivée"], bins=50, kde=True)
```
- **Histogram** shows how delay values are distributed
- `bins=50` divides the data into 50 buckets
- `kde=True` adds a smooth curve showing the distribution shape
- **Insight**: Most trains have 2-10 minute delays; extreme delays (15+ min) are rare

#### Visualization 2: Top 10 Most Delayed Stations
```python
top_delayed = df.groupby("Gare de départ")["Retard moyen..."].mean().sort_values(ascending=False).head(10)
sns.barplot(x=top_delayed.values, y=top_delayed.index)
```
- `groupby("Gare de départ")` groups data by departure station
- `.mean()` calculates average delay for each station
- `.sort_values(ascending=False).head(10)` gets the top 10 worst stations
- **Bar plot** shows which stations have the highest average delays
- **Insight**: International routes (Barcelona, Madrid) have higher delays

#### Visualization 3: Delays Over Time
```python
yearly_delay = df.groupby("year")["Retard moyen..."].mean()
plt.plot(yearly_delay.index, yearly_delay.values, marker="o")
```
- Groups data by year and calculates average delay per year
- **Line plot** shows how delays changed over time
- `marker="o"` adds dots at each data point
- **Insight**: Delay trends show impact of strikes, weather, and infrastructure changes

#### Visualization 4: Delay Causes Breakdown
```python
plt.pie(cause_means.values(), labels=cause_means.keys(), autopct="%1.1f%%")
```
- Calculates average percentage for each delay cause
- **Pie chart** shows proportion of delays caused by different factors
- `autopct="%1.1f%%"` displays percentages on the chart
- **Insight**: Infrastructure and external causes are the biggest contributors

## Key Concepts for Beginners

### What is a DataFrame?
Think of it as an Excel spreadsheet in Python. It has:
- **Rows**: Each row is one record (e.g., one route in one month)
- **Columns**: Each column is a variable (e.g., departure station, delay time)
- **Cells**: Individual values at the intersection of rows and columns

### What is NaN?
- Stands for "Not a Number"
- Represents missing or invalid data
- Like an empty cell in Excel

### What is "groupby"?
- Groups rows that have the same value in a column
- Example: `df.groupby("year")` creates groups for 2018, 2019, 2020, etc.
- Then you can calculate statistics (mean, sum, count) for each group

### What is "median" vs "mean"?
- **Mean** = average (add all values, divide by count)
- **Median** = middle value when sorted
- Median is better for data with outliers (extreme values)
